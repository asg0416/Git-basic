Git 코드 정리

add 는 임시저장, commit은 완전 저장

로컬편

  • 다루고 싶은 폴더로가서 git init을 한다(주의사항: master표시 보이면ㄴㄴ, clone했을때도 하지말자 자동으로 댐
  • 실수로 만들었따 =>ls -al로 확인하고 rm -rf .git (remove r은 폴더지울때 쓰는거, f는 강제로 다 지우는거)
  • git status로 항상 상태 확인, 습관적으로
  • git add . // git add 파일이름
  • git status
  • git commit -m "오늘 여기(뭐까지)했따"
  • add , commit 이런거 반복

협업편 - remote

  • git remote add 이름(origin) 저장소주소.git : 깃허브랑 연결
  • git remote -v : 저장소 이름, 저장소 주소 화가인
  • git push (-u: 디폴트 설정해놓기) origin(목적지 이름) master(우리 현재일하는곳)
  • 만약 reject 및 빨간 글씨 뜬다.
  • git pull origin master
  • pull(다른사람이 한거있으면반영하고 시작하자)
  • add, commit(일하는중) push(업로드), pull(그새 누가 작업했나?)

협업편 - clone(제일 처음 레포만든 사람빼고 나머지)

  • git clone 주소.git
  • git init 하지말자. git remote로 확인해보면 연결되어있음ㅇ
  • add, commit 일하는중,..
  • push(업로드), pull(똉겨오기)
